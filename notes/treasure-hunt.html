What is SOS

  • Platform for Open Source Software users
    • Support with stuff like Linux, IRCs etc.
    • Sharing cool stuff
  • Propaganda for the Open Source Movement
    • Trying to get people to switch to Linux
    • Cool Demonstrations with Open Source software
    • Collaborations with Cultural Clubs
    • Creating a more accessible image for FOSS
  • Helping other tech clubs
    • Providing basic knowledge of tools like git
    • Creating a stronger tech culture

OCR

  • Use OCR on an image to find the number of occurances of a word.
    • There are many FOSS programs available for the same

Examining the Source Code

  • Try out utilities to find out which one does not behave as expected, and crawl through its code in a designated src directory to find the next clue
    • For example, if cp is moving files, check the files in src/cp/

Audio-related tasks

  • Words are hidden in an audio clip's spectrum
    • Analyze it with Audacity
  • Reverse the audio clip to hear the word

Image-related tasks

  • Words hidden in an image (steganograph)
    • steghide
  • Words hidden in PNG metadata

Port-scanning

  • Scan local open ports (ss, netstat) and get a clue

Q. How will you string together clues?

  • One clue leads to the next
    • More exciting
    • One feature not working will break the chain
  • Collect all or as many clues as you can
    • Leaderboard based on how far you get
  • Minimum number of clues to progress to finals
    • Finals can have just one quest
  • Best of both worlds- multiple subquests
    • Finishing Quest 0 will give the problem for Quest 1

Q. Where and how will we implement it?

  • Computer Lab PCs running it locally
    • Set it up with a script that downloads and sets things up as needed
    • USB LiveCDs loaded with all the stuff we need
      • Lots of flash drives will be needed
      • We could have participants bring their own flash drives but it will be hard to convince them
  • Virtual Machines
    • Online VM
      • Server costs thru the roof
      • Setting up multiple user accounts will be hard
    • LiveCD that they'll run on the PC
      • Not sure if Lab PCs can handle VirtualBox/QEMU
  • Player's own PC
    • If we do UNIX-related stuff we'll have to make them use a VM
    • It will be harder to regulate foul play
    • How will we share questions if we don't use a VM?
      • One giant zip file containing all problems
      • Each hint will be a link to a filehost
        • ex. catbox.moe/<clue> -> next puzzle in subquest
